# 三体文明游戏开发文档

## 项目概述

《三体文明》是一款基于刘慈欣《三体》世界观的三维星际经营模拟游戏。玩家需要在乱纪元中建立文明，发展科技，应对三体世界的不稳定环境。

### 技术栈
- **引擎**: Pygame 2.x + OpenGL (3D渲染)
- **语言**: Python 3.9+
- **配置**: YAML
- **数据**: JSON

## 模块使用指南

### ScreenManager 使用

```python
from ui.screen_manager import ScreenManager, ScreenType

# 获取单例实例
manager = ScreenManager()

# 注册界面
manager.register_screen(ScreenType.SETTINGS, settings_screen)

# 切换界面（自动压栈）
manager.switch_to(ScreenType.SETTINGS)

# 返回上级界面
manager.go_back()  # 或指定 fallback: go_back(ScreenType.INITIAL_MENU)

# 清空栈并切换（用于退出到主菜单）
manager.clear_stack_and_switch(ScreenType.INITIAL_MENU)
```

### 创建新界面

```python
from ui.screen_manager import Screen, ScreenType

class MyScreen(Screen):
    def __init__(self, screen_manager, screen):
        super().__init__(screen_manager, screen)
        self.setup_ui()

    def setup_ui(self):
        """初始化UI组件"""
        pass

    def on_enter(self, previous_screen=None, **kwargs):
        """进入界面时调用"""
        super().on_enter(previous_screen, **kwargs)

    def update(self, dt):
        """每帧更新"""
        super().update(dt)

    def handle_event(self, event):
        """处理输入事件"""
        return False

    def render(self, screen):
        """渲染界面"""
        pass
```

### 设置系统使用

```python
from ui.settings_screen import GameSettings

# 创建设置实例
settings = GameSettings()

# 修改设置
settings.fullscreen = True
settings.master_volume = 0.8

# 保存到文件

# 从文件加载
settings.load_from_file("data/settings.json")
```

## 关键修复记录

### 设置界面返回问题修复 (2026-04-12)

**问题描述：**
- 从设置界面无法正常返回主菜单/游戏菜单
- 暂停后无法正确返回游戏

**修复内容：**

1. **settings_screen.py - on_back() 方法：**
```python
# 修复前
def on_back(self):
    self.screen_manager.go_back()

# 修复后  
def on_back(self):
    from .screen_manager import ScreenType
    self.screen_manager.go_back(fallback_screen=ScreenType.INITIAL_MENU)
```

2. **settings_screen.py - on_enter() 方法：**
```python
# 添加屏幕引用更新
super().on_enter(previous_screen, **kwargs)
self.screen = pygame.display.get_surface()  # 新增
self.rect = self.screen.get_rect()         # 新增
self.setup_ui()
```

3. **game_menu.py - on_resume() 方法：**
```python
# 修复前
def on_resume(self):
    self.screen_manager.go_back()

# 修复后
def on_resume(self):
    from .screen_manager import ScreenType
    self.screen_manager.go_back(fallback_screen=ScreenType.MAIN_SCREEN)
```

**修复原理：**
- 使用 `fallback_screen` 参数确保在栈为空时也能正确返回
- 在 `on_enter()` 中更新屏幕引用，确保窗口大小变化后UI正确渲染
- `go_back()` 优先使用栈顶界面，栈为空时使用 fallback

### 连续演化与环境UI适配重构 (2026-04-14)

**问题描述：**
- 游戏模拟仅在打开星图界面时运作，返回主界面后会强制暂止。
- 系统缺乏温度、辐射等真实的物理参数反馈。
- UI组件（主界面标题/计时、星图操作指南面板、HUD内容区域）在不同尺寸/全屏显示器下出现严重缩放异常与重叠。
- 时间缩放倍速调整后，显示的流逝天数和物理演算脱节。

**修复内容：**

1. **统一的后台物理演算体系：**
   - 清除 `starmap_view.py` 的 `on_exit` 强制暂停调用。允许游戏在处于 2D 主界面时，后台继续运行 `simulator.update(dt)` 并产生物理反馈事件。
   - 修正 `simulator.py` 中 `time` 属性累积：绑定统一的 `time_scale` 倍数，与真实的星体运动速度完全匹配。

2. **UI 自适应相对比例渲染 (Responsive Scalability)：**
   - 全面抛弃如固定的 `Y = 45` 或魔法常数大小的绝对屏幕坐标逻辑。
   - 大规模引进了动态基准系数 `scale = min(width / 1280, height / 720)`，确保字体、高度及布局位置均依据当前窗口比例相对生成，使得标题与数值文本永不重叠，各类浮窗在大分辨率显示器下依然清晰可用。

3. **环境动态深度拓展与重新布局：**
   - **`game/environment.py`**: 实装依附非线性距离平方反比公式与质量所映射的「辐射」数值（mSv）及「环境温度」数值（℃）。
   - **`ui/main_screen.py`**: 在主操作区将面板切分为资源、文明、环境信息及行动 4 个相对等比的大板块。
   - **`render/ui.py`**: 清除星图内无关的建造信息，统一顶层环境监控 HUD (`update_hud`) 界面参数逻辑，保持了平滑与专注。

### 星图行星锁定视角功能 (2026-04-20)

**功能描述：**
- 在星图界面顶部新增"锁定行星"按钮，点击后相机切换为**轨道模式**，围绕行星（三体世界）进行观察。
- 点击"解锁"按钮恢复自由视角模式。

**两种相机模式操作对比：**

| 操作 | 自由模式 | 锁定行星模式 |
|------|---------|------------|
| 鼠标拖拽 | 旋转视角方向 | 围绕行星旋转观察角度 |
| W/S | 向前/后移动 | 拉近/拉远行星 |
| 滚轮 | 沿视角方向前进/后退 | 拉近/拉远行星 |
| A/D | 左/右平移 | 水平方向环绕行星 |
| Q/E | 上/下移动 | 垂直方向调整俯仰 |

**技术实现：**

1. **`render/camera.py`**：新增球坐标轨道相机系统（`set_lock_target`, `orbit_rotate`, `orbit_zoom`, `update_orbit`）。锁定时相机使用 `(orbit_distance, orbit_yaw, orbit_pitch)` 球坐标围绕目标，每帧自动跟踪行星最新位置。
2. **`ui/starmap_view.py`**：新增锁定按钮及 `planet_locked` 状态。输入处理根据锁定状态分发到不同逻辑。帮助面板根据模式动态切换操作说明。
3. **`render/ui.py`**：罗盘（Compass）在锁定模式下显示 `[LOCKED]` 状态指示和轨道距离信息，边框变为金色。

### 实体底层解构与 UI 统一重绘重构 (2026-04-20)

**问题描述：**
- 游戏主界面存在“资源”和“文明”数值重复（人口数据分别显示渲染），并且包含赘余的“行动”面板。
- 底层资源操作极度离散，科技树和政策的锁定判断只通过传递一个巨大的状态字典，不利于政策直接干预和消耗资源（如消耗食物降低稳定性）。
- 由于科技树与政策树在 UI 初始化调用时遗漏了更新后的传参，导致点击解锁或执行政策时产生报错。

**修复与重构内容：**

1. **底层实体与资源重构 (`game/entities.py`, `technology.py`, `policy.py`)：**
   - 移除独立的 `population` 变量。将其直接转化为标准的 `Resource("population", ...)` 类，与金属、能源平级，统一归口。
   - 在 `EntityManager` 新增 `get_resource`, `consume_resource`, `produce_resource` 等健壮的业务底层 API。
   - `Technology` 与 `Policy` 的所有前置判定、执行环节的传参，从 `dict` 彻底变更为直接传入 `EntityManager` 或 `self.simulator.entities` 对象，从而允许“大生育计划”能够立即真正消耗 500 点 `food` 并判定失败。

2. **UI 清理与三区段等比适配 (`ui/main_screen.py`)：**
   - 将主界面原先横向排布四个占位版块删除 `行动`，重构为等比均分的高视界开阔性三面板：`资源总览`、`文明与状态` 和 `环境监控`。
   - 剔除了原先由于尚未实装对应演算而在 UI 上生硬写死的数值增减率装饰虚假字符串（如 `+2/天`、`+0.5%/天`），使得主控制台数据更符合当前真实的模拟快照。

3. **修复崩溃异常：**
   - 彻底修复因为重构过程中传参失效引起的 TechTree 页面与 Policy 页面参数崩溃 (`'dict' object has no attribute 'get_resource'`)。
